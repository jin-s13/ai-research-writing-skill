# 摘要写作指南

## 目标

通过反复做三件事，写出强有力的摘要：

1. 先想清楚摘要逻辑。
2. 遵循下面某一版模板（版本 1/2/3）。
3. 多次修订摘要。

## 写作前问题（重要）

动笔前先回答：

1. 我们解决什么技术问题？为何尚无成熟方案？（重要）
2. 我们的技术贡献是什么？
3. 我们的方法在本质上为何能奏效？
4. 我们提供哪些技术优势与新洞见？（重要）

## 版本 1：挑战 → 贡献

先介绍技术挑战，再用一至两句呈现解决该挑战的技术贡献。

### 结构

1. 任务。
2. 以往方法面临的技术挑战。
3. 用一至两句介绍解决该挑战的技术贡献。
4. 该技术贡献带来的好处。
5. 实验总结。

### 专家要点

1. 围绕我们实际解决的技术挑战讨论前人工作。
2. 贡献句通常只提技术术语/名称；不必逐步解释每个细节。
3. 技术术语须易懂；读者不应感到突兀跳跃。
4. 这种能力对写好摘要非常重要。

## 版本 2：挑战 → 洞见 → 贡献

先介绍技术挑战，再用一至两句呈现解决该挑战的洞见，再用一句呈现实现该洞见的技术贡献。

### 结构

1. 任务。
2. 以往方法面临的技术挑战。
3. 用一句介绍解决该挑战的洞见。
4. 用一至两句介绍实现该洞见的技术贡献。
5. 技术新颖性带来的好处。
6. 实验总结。

### 专家要点

1. 围绕我们实际解决的技术挑战讨论前人工作。
2. 用一句清晰的话介绍洞见。
3. 实现句通常只提技术术语/名称；不必逐步解释每个细节。
4. 技术术语须易懂；阅读中勿造成跳跃感。
5. 这种能力对写好摘要非常重要。

版本 2 本地引用：

1. `references/examples/abstract/template-b.md`

## 版本 3：多项贡献

版本 3：当存在多项技术贡献时，将每项贡献与其技术优势一并描述。

### 结构

1. 任务。
2. 如需要，用一句对比前人方法。
3. 贡献句 1 + 技术优势。
4. 贡献句 2 + 技术优势。
5. 贡献句 3 + 技术优势。
6. 实验总结。

### 专家要点

1. 多项技术贡献时，将每项贡献与其技术优势一并描述。
2. 用一句同时表达「贡献 + 优势」的能力对写好摘要非常重要。

## 摘要质量检查清单

1. 读者能否一次读通任务、挑战、洞见/贡献与结果？
2. 所有主要主张是否都有实验支撑？
3. 技术名称是否自洽、可读？
4. 是否有句子混杂过多信息？
