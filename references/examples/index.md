# Example Bank Index

Use this folder only when concrete writing examples are useful after choosing a pattern from `references/example-bank.md`, `references/abstract-introduction.md`, or `references/section-writing.md`.

The main `references/` files define workflow and rules. This folder keeps only concrete examples, annotated samples, and anti-patterns. Reuse sentence logic and paragraph structure, not exact wording.

## Abstract Examples

- `abstract/challenge-insight-contribution.md`: challenge -> insight -> contribution abstract examples.
- `abstract/multiple-contributions.md`: abstract example for papers with multiple technical contributions.

## Introduction Examples

- `introduction/open-with-challenge.md`: opening paragraph that exposes the technical challenge early.
- `introduction/existing-task-challenge.md`: challenge chain for an established task with existing methods.
- `introduction/traditional-insight-backed.md`: use a traditional insight to motivate a modern contribution.
- `introduction/novel-task-decomposition.md`: decompose a novel task into concrete requirements or challenges.
- `introduction/pipeline-one-contribution.md`: introduce one contribution with multiple advantages.
- `introduction/pipeline-two-contributions.md`: introduce two linked contributions.
- `introduction/pipeline-new-module.md`: add one new module to an existing pipeline.
- `introduction/pipeline-observation-driven.md`: introduce a method from one key observation.
- `introduction/anti-pattern-abstract-only.md`: anti-pattern where the Introduction stays too abstract and hides the actual pipeline.

## Method Examples

- `method/neural-body-annotated.md`: annotated method-section roles for Neural Body.
- `method/instant-ngp-annotated.md`: annotated module design example for Instant-NGP-style hash encoding.
- `method/overview-template.md`: concrete Method overview paragraph template with examples.
- `method/module-triad-example.md`: full module-triad example showing motivation, design, and technical advantage.

## Related Work Examples

- `related-work/line-of-work-paragraph.md`: concrete paragraph pattern for summarizing one research line and its remaining gap.
- `related-work/close-work-positioning.md`: neutral positioning pattern for close or concurrent work.

## Loading Guidance

1. Load `references/example-bank.md` first for compact patterns.
2. Load one example file only when the agent needs a concrete writing sample.
3. Prefer the narrowest matching example; do not load the whole examples folder.
4. Keep claims and citations tied to the user's paper evidence rather than copying example claims.
